# 2026-07-23 사용자 요청 및 개선사항 이력

## 1. 개요
본 문서는 2026년 7월 23일 진행된 **헬스케어 포털 메뉴설정 > 메뉴구성 및 노출관리** 화면 내 `교보다솜케어 & 교보다솜헬스케어` 공통(슈퍼) 사이트의 **2단계 사이트 정보 및 등급 수정 팝업** 개편 작업에 대한 요청사항 및 조치 내역을 정리한 이력 관리 문서입니다.

---

## 2. 주요 요구사항 및 수정 내역

### 2.1 공통 사이트(슈퍼 사이트) 개념 도입 및 안내 문구 배치
- **요청사항**: 분기사이트를 제작하지 않는 고객사들이 공통으로 이용하는 사이트(`교보다솜헬스케어`)를 슈퍼사이트 개념으로 개편하고 안내 문구 표시.
- **수정사항**:
  - 중복 안내를 방지하기 위해 팝업 최상단 타이틀 영역의 이중 설명을 제거하고, 사이트명 입력란 옆 `공통 이용사이트` 배지 우측에 안내 문구를 적용했습니다.
  - **안내 문구**: `이 사이트는 분기사이트를 생성하지 않는 고객사들이 공통으로 이용하는 사이트입니다.`

### 2.2 2단계 설정 방식 개편 (고객사 복수 선택 & 고객사별 등급 설정)
기존 단일 등급 Shuttle Box 방식 대신 2단계 설정 프로세스로 전환했습니다.

1. **1단계: 이용 고객사 복수 선택 (Shuttle Box)**
   - 전체 고객사 목록에서 실시간 검색 및 체크박스 선택 가능.
   - `[ 추가 ▶ ]` 및 `[ ◀ 제거 ]` / `[ ✕ ]` 버튼으로 선택된 고객사 목록 관리.
   - 하단 `[ 선택 완료 ]` 버튼 클릭 시 2단계로 부드러운 스크롤 이동 연동.
2. **2단계: 고객사별 등급 설정 (고객사 검색 & 드롭다운 & Shuttle Box)**
   - 20~30개 이상의 고객사를 효율적으로 관리하기 위해 탭 방식 대신 **고객사 검색 입력(`고객사 검색...`) + 드롭다운 선택 방식**(`설정할 고객사: 전체 / 고객사별`)을 적용.
   - 키워드 입력 시 드롭다운 목록이 실시간 필터링되어 원하는 고객사를 빠르게 검색하고 선택할 수 있도록 연동.
   - 고객사를 전환하더라도 각 고객사별 서비스 등급 설정 내용(`clientTierMap`)이 각각 독립적으로 유지·저장.
   - 하단 `ⓘ 안내` 박스(고객사별 설정 유지 및 전체 선택 고객사 최소 1개 등급 지정 필수) 및 `저장 시 유의사항` 점선 블록 탑재.

### 2.4 페이지 이탈 시 최상단 [변경사항 저장] 미클릭 경고 알림 구현
- **요청사항**: 메뉴설정 페이지에서 변경사항 적용 후 최상단 `[변경사항 저장]` 버튼을 누르지 않은 채 이탈하려 할 경우 경고 알림을 띄움.
- **수정사항**:
  - 변경 상태 관리 변수(`isMenuSettingsDirty`)를 도입하여 팝업 수정, 메뉴 추가/삭제/변경, 입력 폼 수정 발생 시 자동 활성화.
  - 최상단 `[변경사항 저장]` 버튼 클릭 시 변경사항을 영구 저장(`localStorage`)하고 변경 상태 초기화.
### 2.5 분기사이트 필수 등록 요건: 포털 URL명(분기 경로) 입력란 UI 및 유효성 검사
- **요청사항**: 헬스케어포털 메인 도메인 URL 뒤에 붙는 분기사이트 경로명(예: `branch-a`)을 등록할 수 있는 전용 입력란 배치 및 필수 검증.
- **수정사항**:
  - `헤더 및 브랜드 설정` 내 `사이트명` 영역을 2열 그리드로 나누고, 우측에 **`https://healthcare.co.kr/` [ `branch-a` ]** 형태의 프리픽스 결합형 입력 필드 구성.
  - 일반 분기사이트는 `포털 URL명(분기 경로)` 미입력 시 저장 클릭 시 경고 알림(`포털 URL명(분기 경로)은 분기사이트의 필수 등록 요건입니다. (예: branch-a)`)을 띄우고 저장 프로세스를 차단.
  - 공통(슈퍼) 사이트(`교보다솜헬스케어`)의 경우 포털 URL 입력이 제외되므로 비활성화(Disabled) 및 안내 문구(`(공통 사이트는 포털 URL 입력 제외)`) 표시.
### 2.6 사이트 운영 상태 설정 (운영 / 미운영) 및 미운영 페이지 차단 화면
- **요청사항**: 사이트 설정 제일 하단에 `운영 / 미운영` 설정 옵션 추가. 미운영 시 URL 접속 차단 및 브릿지 미노출 (단, 관리자에서는 기존 설정 조회·수정 유지).
- **수정사항**:
  - `포털 메뉴설정` 최하단에 **`사이트 운영 상태 설정`** 카드를 신설하고 2가지 옵션 라디오 카드를 배치:
    - **`● 운영`**: URL 접속 가능, 대상 고객 브릿지 페이지 노출
    - **`○ 미운영`**: URL 직접 접속 차단, 브릿지 페이지 미노출 *(※ 관리자에서는 기존 고객사·등급·메뉴 설정을 그대로 조회·수정 가능)*
  - 관리자 사이트 탭 영역에 미운영 사이트 식별용 **`[미운영]`** 빨간색 배지 노출.
  - 사용자 포털/모바일 앱에서 `미운영` 상태 사이트 직접 접근 시 접속 차단 화면 노출:
    - **안내 타이틀**: `현재 운영하지 않는 헬스케어 사이트입니다.`
    - **안내 서브 문구**: `이용 관련 문의는 고객센터로 문의해 주세요.`

### 2.7 '헬스케어 포털 메뉴설정' 클릭 시 사이트 관리 목록 화면 및 진입 플로우 개편
- **요청사항**: 어드민에서 '헬스케어 포털 메뉴설정'을 클릭하면 기존 메뉴 구성 및 노출관리가 뜨기 전 기존 생성 사이트 리스트(사이트 관리 화면)가 노출되고, 신규사이트 생성 버튼 배치. 관리 컬럼의 `수정` 또는 `신규사이트 생성` 클릭 시 기존 메뉴 구성 및 노출관리 편집 화면으로 진입.
- **수정사항**:
  - **[사이트 관리] 목록 화면 구현**:
    - **상단 현황 요약**: `전체 고객사 수` (24개) 및 `전체 사이트 수` (7개) 대형 수치 카드 배치.
    - **검색 & 필터 패널**: 고객사명, 사이트명, 사이트 구분(전체/공통/분기), 운영상태(전체/운영/미운영), 적용등급, URL 6대 검색 옵션 및 `초기화`, `검색` 버튼 장착.
    - **목록 헤더 & Action**: `총 N건` 실시간 카운트, `+ 신규사이트 생성` 버튼, `엑셀 다운로드` 버튼 연동.
    - **사이트 목록 테이블**: 고객사명(`교보다솜케어`로 정명 표기), 사이트명, **유형**(`공통`/`분기` 배지), 접속 URL(기본: `https://kyobodasomcare.com/health`, 분기: `https://kyobodasomcare.com/health/분기명`), 적용등급(`VIP 외 2개 ⓘ` 형태 및 ⓘ 클릭 시 전체 등급 팝업 노출), 운영상태(운영 green 뱃지 / 미운영 gray 뱃지), 관리(`수정` 버튼 - 한 줄 정렬 및 여백 확보) 구성.
  - **진입 및 복귀 플로우 구축**:
    - 사이트 목록 테이블 내 **`수정`** 버튼 클릭 시 해당 사이트의 `메뉴 구성 및 노출관리` 편집 화면으로 진입.
    - **`+ 신규사이트 생성`** 모달 개편: **1순위로 대상 고객사 선택 Dropdown**을 최상단에 배치하고, 등급 Shuttle Box 모달을 제거하여 `[1. 대상 고객사 선택]`, `[2. 사이트명]`, `[3. 포털 URL명]` 입력만으로 간편히 생성 후 바로 `메뉴 구성 및 노출관리` 편집 화면으로 이동하도록 전면 간소화.
    - 메뉴 편집 화면 좌상단에 **`← 사이트 목록`** 버튼을 배치하여 언제든지 목록 화면으로 복귀 가능하도록 UX 완성.
