# 헬스케어포털 변경 이력 (Changelog)

이 문서는 헬스케어포털 프로젝트의 주요 변경 사항, 기능 추가 및 버그 수정 내역을 관리합니다.

## [2026-05-15]
### 건강상담 및 이력 페이지 UI/UX 개편
- **건강상담 신청 가이드 고도화**:
    - 전화상담/온라인문의 유형별 맞춤형 가이드 문구 동적 노출 시스템 구현
    - '상담 신청 정보 입력' 영역 하단에 유형별 주의사항 및 프로세스 안내 박스 추가
- **상담 이력 리스트 UI 전면 개편**:
    - 문의유형, 등록일, 내용, 희망일시, 상태를 포함한 전문적인 리스트 레이아웃 구축
    - 상태별 배지 적용 (신청: Black, 완료: Green, 취소: Gray)
    - 아코디언(Expandable) 방식의 상세 보기 기능: 질문(Q.)과 답변(답변내용 배지 + 파란색 텍스트) 시각적 강화
- **메뉴 판별 로직 개선**: 내부 ID 방식에서 명칭(Label) 기반 키워드 매칭 방식으로 변경하여 동적 메뉴 호환성 확보
- **시스템 안정화**: `app.js` 내 문법 오류 및 중복 구문 제거를 통한 렌더링 무결성 확보

## [2026-05-13]
### 기능 고도화 및 브랜드 관리 시스템 도입
- **3단계 계층형 메뉴 시스템 (3-Level Menu)**:
    - 1depth > 2depth > 3depth 구조의 재귀적 트리 메뉴 관리 기능 구현
    - 각 메뉴 노드별 독립적인 노출 여부(Show/Hide) 및 명칭(기본명/고객사별 명칭) 설정 가능
    - 대규모 메뉴 구조를 한눈에 파악하고 편집할 수 있는 확장형 관리 UI 적용
- **프리미엄 메가메뉴 (Mega-Menu) 구현**:
    - 사용자 포털 헤더 호버 시 3단계 계층 구조가 한눈에 펼쳐지는 그리드 기반 메가메뉴 레이아웃 적용
    - 부드러운 애니메이션 및 직관적인 다단 컬럼 네비게이션 제공
- **사이드바 아코디언 네비게이션**: 3단계 계층 구조를 지원하며, 선택 메뉴의 하위 항목이 자동 확장되는 직관적인 트리뷰 구현
- **1:1 온라인 문의 시스템**: 사용자 개인 질문과 전문가 답변을 매칭하는 비공개 문의 게시판 및 어드민 대응 시스템 구축
- **헤더 및 브랜드 통합 관리 (Branding)**:
    - **로고 업로드 시스템**: SVG 및 이미지 파일을 관리자 페이지에서 직접 업로드하여 실시간 반영 (Base64 저장)
    - **2포인트 컬러 시스템**: 포인트 컬러 1(버튼/로그인 배경)과 포인트 컬러 2(메뉴/CS번호)로 구성된 직관적인 테마 관리
    - **HEX 코드 직접 입력 및 미리보기**: 컬러 피커 대신 HEX 코드를 직접 입력하고 즉시 색상을 확인하는 프리뷰 기능 탑재
- **동적 푸터(Footer) 및 정책 관리**:
    - 고객사명, 고객사 링크, 서비스 제공사명(교보다솜케어), 제공사 링크 동적 설정 기능
    - 공식 서비스 고지 문구(Disclaimer) 및 법적 필수 링크(개인정보처리방침, 이용약관) 연동
- **글로벌 브랜딩 기본값(Default) 수립**:
    - 모든 신규 고객사 포털에 대해 '교보다솜케어' 브랜딩을 기본값으로 강제 적용하여 초기 설정 편의성 및 브랜드 통일성 강화

## [2026-05-12]
### 기능 추가 및 구조 변경
- **추가 메뉴 동적 생성 기능**: 관리자 포털에서 고객사별 기본 메뉴 외에 새로운 커스텀 메뉴를 동적으로 생성 및 삭제 기능
- [관리자] 건강정보 관리 메뉴 신설 및 WYSIWYG 에디터 구현
- [관리자] 건강검진 관리 시스템 고도화
    - 하위 메뉴 분리: 제휴 병원 관리, 검진 패키지 관리, 검진 항목 관리
    - 검진 항목 마스터(Pool) 관리 시스템 도입으로 패키지 구성 편의성 극대화
    - 병원별 예약/정산 담당자 분리 관리 및 포털 노출 정보 강화
- **등급별 메뉴명 설정 기능**: 추가된 커스텀 메뉴의 이름을 전체 사용자 공통으로 설정하거나, 고객사 내 사용자 등급(Tier)별로 각각 다른 이름이 노출되도록 세분화하여 설정하는 UI 구현
- **사용자 포털 다이내믹 라우팅 및 렌더링 연동**: 로그인한 사용자의 등급 정보와 관리자가 설정한 커스텀 메뉴 데이터를 매칭하여 사용자 포털 GNB에 동적 메뉴가 즉시 반영되도록 로직 연동
