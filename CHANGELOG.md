# 헬스케어포털 변경 이력 (Changelog)

이 문서는 헬스케어포털 프로젝트의 주요 변경 사항, 기능 추가 및 버그 수정 내역을 관리합니다.

## [2026-05-20]
### 건강검진 우대예약 프리미엄 랜딩 페이지 및 카드 UI 구축
- **건강검진 우대예약 전용 프리미엄 화면 구현**:
    - '건강검진 예약' 메인 메뉴 하위에 '건강검진 우대예약' 서브메뉴 추가 및 클릭 시 독립적인 고품질 UI(히어로 배너, 검색 엔진, 2단 병원 리스트 등)가 노출되도록 동적 렌더링 로직 추가.
    - 화면 최하단에 '검진 비용 병원 직접 결제' 안내 배너 추가.
- **다차원 복합 검색 필터 (희망일/희망검사) UI 신설**:
    - 기존 지역/병원명 검색 외에 **희망일자(달력 연동)** 및 **희망검사(다중 선택 드롭다운)** 필터를 상단에 통합한 2단 레이아웃 구현.
    - 검사 항목 다중 선택 시 직관적인 태그(Tag) UI가 동적으로 생성되도록 시각적 피드백 강화.
- **위치정보 수집 동의 커스텀 모달 및 어드민 로그 연동**:
    - '내 주변 검색' 클릭 시 프리미엄 UI가 적용된 커스텀 모달을 띄워 위치정보 제공 동의/거부 프로세스 연동.
    - 관리자 포털(어드민) 좌측 대시보드 하위에 '위치정보 동의 로그' 메뉴를 신설하고, 사용자별 동의 여부, 목적, IP, 기기 환경 등 10가지 상세 데이터를 모니터링할 수 있는 조회 테이블 구축.
- **URL 인코딩 매칭 예외 처리 (Robust Routing)**:
    - 어드민에서 설정한 한글 메뉴 ID("건강검진 우대예약")가 브라우저 URL 해시(hash)로 넘어올 때 발생하는 인코딩(%EA...) 이슈를 해결하기 위해, `pageTitle.includes()` 및 `decodeURIComponent` 기반의 유연한 라우팅 조건식 적용.
- **병원 리스트 카드 미니멀리즘 디자인 최적화**:
    - 직관성을 해치는 기존 뱃지 및 문구(우대 뱃지, 우대율 XX% 텍스트, 종합검진 아이콘)를 일괄 삭제.
    - 병원 주소 아래 영역에 '2026년 대장내시경 마감'과 같이 긴급/특이사항을 파란색 굵은 텍스트로 가독성 있게 노출하는 전용 정보 영역 신설.
- **병원 상세 및 검진항목 다중 탭 모달 구현**:
    - 병원 리스트에서 '병원정보' 및 '검진항목' 버튼 클릭 시, 하나의 통합된 프리미엄 모달 창에서 독립적인 다중 탭 구조로 렌더링.
    - '검진항목' 탭 내에 기본/선택/추가 검사항목의 3단 서브 탭을 신설하고, 포털 테마 컬러(Blue)가 적용된 병합 테이블(이중 Rowspan) 구현.
- **예약 일정보기(달력) UI 고도화 및 스마트 탐색 알고리즘**:
    - 고객이 휴일, 예약 마감, 예약 가능 현황 및 특정 검사(대장내시경 등) 마감 여부를 한눈에 파악할 수 있는 시각적 분리 캘린더(파란색/빨간색/취소선) 레이아웃 탑재.
    - 당월 전체가 마감된 경우 예약 가능한 일자가 존재하는 다음 월로 달력을 자동 스킵(Fast-forwarding)하는 알고리즘 구현.

### 예약 진행 단계(STEP 3) 및 이력 조회 기능 고도화
- **STEP 3 검진항목 선택 UI 전면 개편**:
    - **기본검사**: 패키지별 지정 항목을 아코디언으로 펼쳐볼 수 있으며, 접힌 상태에서는 '장비검사명'만 간략히 노출되도록 개선.
    - **선택검사**: 선택그룹별 제목 노출 및 펼침 시 개별 체크박스 생성. 사용자의 혼란을 줄이기 위해 '최대 N개 선택' 문구를 'N개 선택' 필수 조건으로 변경.
    - **추가검사**: 자유로운 항목 추가 가능성 및 비용 합산 규칙에 대한 상세 안내 문구 상단 배치.
    - **시각적 가독성 개선**: 선택검사 및 추가검사의 설명/비고란을 색상 구분 및 하단 소형 텍스트로 표기하고, 약관 동의 영역을 다음 버튼 바로 위로 이동시켜 UX 동선 단축.
- **최근 5년 과거검진신청이력 모달 연동**:
    - 우측 상단 버튼 클릭 시 팝업되는 고급 모달을 통해 과거 5년(2021~2025년) 확정 이력 연도별 조회 지원. (취소 이력 제외 필터링 안내 포함)
    - 본인 신청 및 가족 신청 탭을 제공하며, 양쪽 모두 기본/선택/추가 검사항목이 명확히 구분되는 3단 병합 테이블 레이아웃을 동일하게 적용하여 정보 통일성 확보.

### 시스템 최적화 및 레거시 파일 정리
- **사용되지 않는 불필요한 JS 파일 검출 및 분리**:
    - `index.html`과 `admin.html`의 소스 참조를 분석하여, 핵심 파일(`app.js`, `admin.js`) 외의 과거 디버깅 및 인코딩 과정에서 파생된 6개의 찌꺼기 임시 파일들을 정확하게 식별.
## [2026-05-19]
### 어드민 메뉴 노출 관리 고도화 및 공통 메뉴 동기화 아키텍처 도입
- **공통 메뉴 구조 통합 관리 및 개별 고객사 노출 정책 수립**:
    - 메뉴 구조의 일관성을 위해, 어드민에서 특정 고객사의 사이트를 편집하여 메뉴를 생성/수정(명칭 변경)/삭제 시, **전체 고객사의 사이트 메뉴 데이터에 동일한 계층으로 실시간 동기화**되는 전역 동기화 기능 구현.
    - **신규 메뉴 추가 시 노출 제어 정책 적용**: 새로운 메뉴(최상위 또는 하위)를 추가하면, **현재 작업 중인 해당 고객사의 사이트에만 노출(`isVisible: true`)**로 활성화되고, **타사 모든 고객사의 사이트들에는 `isVisible: false`로 미노출 기본값**이 안전하게 할당되도록 제어 로직 설계.
    - **백그라운드 자동 복구 및 자가 치유(Auto-Healing Migration)**: 로컬 스토리지에 파편화되어 있을 수 있는 기존 데이터를 방어하기 위해, 포털/어드민 로드 시 **교보생명(kyobo)의 메뉴 구조를 마스터 템플릿으로 삼아 전체 고객사의 메뉴 트리를 실시간으로 강제 보정/일치**시키는 `syncAllClientMenus` 백그라운드 데이터 연동 레이어 구현.
- **상/하위 메뉴 노출 제어 연동 기능 추가**:
    - 어드민 메뉴 구성 및 노출 관리 화면에서 상위 카테고리 메뉴의 노출 스위치를 비활성화(미노출)할 경우, 하위 서브메뉴(2depth 및 3depth)의 노출 여부도 재귀적으로 탐색하여 자동으로 모두 비활성화(미노출)되도록 개선.

### 건강검진 예약 메뉴의 프리미엄 정적 가이드 페이지 개편
- **단일 레벨 메뉴화 및 정적 정보 카드로 구조화**:
    - 기존 '건강검진 예약' 하위의 복잡한 이원화 메뉴 구조를 걷어내고, 직관적인 **단일 레벨 탑메뉴(Flat Structure)**로 전환.
    - 입력창 폼 및 접수 버튼을 완전히 제거하여 사용자 예약 경로의 혼선을 근본적으로 방지하고, 깔끔하게 설계된 전문가 설계/비교 선택에 대한 **순수 정보 안내 카드로 전면 개편**.
    - 카드 하단의 보조 안내 파란색 뱃지 영역까지 모두 삭제하여, 4대 핵심 혜택 그리드와의 비주얼 시너지를 극대화한 미니멀리즘 디자인 완비.
- **White-Label 브랜드 중립성 확보**:
    - 모든 임직원 고객사가 공통으로 활용하는 메인 페이지임을 고려하여, 본문 첫 머리의 특정 제공사 명칭 접두사인 **"교보생명 헬스케어는"** 문구를 완벽하게 제거하고 중립적인 공통 설명체(`고객의 건강 상태와 필요에 따라...`)로 대체.

## [2026-05-18]
### 동적 멀티 사이트 빌더 및 메뉴별 등급 권한 제어 시스템 (Fine-Grained RBAC) 구축
- **동적 멀티 테넌트/멀티 사이트 아키텍처 개편**:
    - 단일 고객사(Client) 하위에 N개의 독립적인 사이트(Site)를 생성, 편집 및 삭제할 수 있는 데이터 모델 설계.
    - 기존 단일 사이트 기반 데이터(menus, themeColor, logoImage 등)를 로드 시 자동으로 **`[기본 사이트 (default)]`**의 구조화된 데이터 세트로 전환/이식하는 무결성 마이그레이션 로직 구현.
- **어드민 가로 탭 사이트 셀렉터 UI 추가 및 사이트 정보/등급 수정 기능 연동**:
    - 고객사 에디터 상단에 직관적인 가로형 탭 형태의 사이트 관리 컨트롤 영역 배치.
    - **`[+ 사이트 추가]`** 버튼 클릭 시, 모달 팝업을 통해 사이트명 지정 및 접속을 허용할 유저 등급군(`mappedTiers`)을 매핑할 수 있는 생성 프로세스 완비.
    - **활성화 탭 클릭 시 정보/등급 수정 연동**: 이미 선택된 사이트 탭(예: 기본 사이트)을 한 번 더 클릭할 경우, 사이트명 및 등급을 수정할 수 있는 **'사이트 정보 및 등급 수정'** 셔틀 박스 모달 팝업(`openEditSiteModal()`)이 즉각 호출되도록 UX 고도화.
    - **실시간 비주얼 인디케이터**: 활성화 탭 옆에 `✏️` 편집 아이콘 및 마우스 오버 시 `클릭하여 사이트명 및 등급 수정` 안내 툴팁이 표기되도록 직관성 강화.
    - **메인 브랜드 설정에 사이트명 입력창 추가**: 어드민 메인 화면의 **'헤더 및 브랜드 설정'** 카드에도 사이트명 입력창(`input-siteName`)을 추가하여 탭 클릭 모달 외에도 메인 편집 폼에서 사이트명을 직접 수정할 수 있도록 활성화.
    - 활성화된 사이트 탭(각 사이트의 포인트 컬러 동적 적용)을 전환할 때마다 3depth 메뉴 트리, 브랜드 에디터(로고, 테마 컬러, CS정보 등)가 해당 사이트의 설정 데이터로 즉시 교체 및 개별 저장.
- **메뉴별 상세 등급 노출 설정 및 신규 사이트 등급 선택의 Shuttle Box (Dual List Box) 전환**:
    - **셔틀 박스 UI 컴포넌트 탑재**: '신규 사이트 추가' 모달 및 '메뉴 노출 조건 설정' 모달 내의 등급 선택 UI를 최신 셔틀 박스(Dual List Box) 형태로 전면 개편.
    - **다차원 실시간 검색 및 다중 선택**: 좌/우 각 패널 상단에 등급 검색창(`filterLeft/Right`)을 배치하여 등급이 많을 시에도 100ms 이내 초고속 필터링 지원, 개별 다중 체크박스를 통한 편리한 추가/제거 기능 구현.
    - **중앙 직관적 액션 컨트롤**: `[추가 ▶]`, `[◀ 제거]` 버튼을 통해 선택된 등급군을 실시간으로 상태 이동 및 동적 렌더링.
    - **데이터 격리 예외 처리**: 사이트 간 완벽한 데이터 독립성(Data Isolation)을 위해, 이미 다른 사이트에 매핑 완료된 등급은 좌측 목록에서 취소선과 함께 비활성화(Disabled) 및 `(다른 사이트 지정됨)` 배지를 표시하고 중복 추가를 원천 차단하여 비즈니스 논리 안정성 극대화.
    - **오브젝트 출력 방지**: 기존 등급 데이터 로드 시 등급 속성 자료형(String/Object)에 따른 `[object Object]` 문자열 출력 문제를 감지하여 완벽히 방어(Fallback 로직 구현 완료).
- **사용자 포털 브릿지 페이지 (Bridge Selection Page) 구현**:
    - 로그인한 사용자가 2개 이상의 사이트에 접근 권한이 있을 경우, 진입 전 맞춤형 포털 목록을 보여주는 **고품격 사이트 선택 브릿지 카드 UI** 구현.
    - 카드 호버 시 상단 테마 컬러 바가 강조되며 부드럽게 떠오르는 그림자(Shadow)와 버튼 전환 효과 등 프리미엄 디자인 설계.
    - 단 1개의 사이트에만 접근 가능한 유저의 경우, 브릿지 페이지를 스킵하고 즉시 해당 포털 메인으로 자동 포워딩하여 사용자 편의성(UX) 극대화.
- **포털 내 실시간 사이트 스위처 Dropdown 적용**:
    - 다중 사이트 접근 권한을 가진 사용자가 포털 내부에서 로그아웃 없이 다른 사이트로 즉시 전환할 수 있도록, 헤더 우측 사용자명 옆에 테마 컬러가 입혀진 **실시간 사이트 셀렉터 Dropdown** 탑재.
- **Fine-Grained 재귀적 메뉴 권한 필터링**:
    - 어드민에서 특정 메뉴에 설정한 노출 등급 세팅값(`menu.exposedTiers`)을 재귀적으로 탐색하여 유저 등급과 비교 분석(합집합 OR 조건).
