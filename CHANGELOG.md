# 헬스케어포털 변경 이력 (Changelog)

이 문서는 헬스케어포털 프로젝트의 주요 변경 사항, 기능 추가 및 버그 수정 내역을 관리합니다.

## [2026-05-18]
### 동적 멀티 사이트 빌더 및 메뉴별 등급 권한 제어 시스템 (Fine-Grained RBAC) 구축
- **동적 멀티 테넌트/멀티 사이트 아키텍처 개편**:
    - 단일 고객사(Client) 하위에 N개의 독립적인 사이트(Site)를 생성, 편집 및 삭제할 수 있는 데이터 모델 설계.
    - 기존 단일 사이트 기반 데이터(menus, themeColor, logoImage 등)를 로드 시 자동으로 **`[기본 사이트 (default)]`**의 구조화된 데이터 세트로 전환/이식하는 무결성 마이그레이션 로직 구현.
- **어드민 가로 탭 사이트 셀렉터 UI 추가 및 사이트 정보/등급 수정 기능 연동**:
    - 고객사 에디터 상단에 직관적인 가로형 탭 형태의 사이트 관리 컨트롤 영역 배치.
    - **`[+ 사이트 추가]`** 버튼 클릭 시, 모달 팝업을 통해 사이트명 지정 및 접속을 허용할 유저 등급군(`mappedTiers`)을 매핑할 수 있는 생성 프로세스 완비.
    - **활성화 탭 클릭 시 정보/등급 수정 연동**: 이미 선택된 사이트 탭(예: 기본 사이트)을 한 번 더 클릭할 경우, 사이트명 및 등급을 수정할 수 있는 **'사이트 정보 및 등급 수정'** 셔틀 박스 모달 팝업(`openEditSiteModal()`)이 즉각 호출되도록 UX 고도화.
    - **실시간 비주얼 인디케이터**: 활성화 탭 옆에 `✏️` 편집 아이콘 및 마우스 오버 시 `클릭하여 사이트명 및 등급 수정` 안내 툴팁이 표기되도록 직관성 강화.
    - **메인 브랜드 설정에 사이트명 입력창 추가**: 어드민 메인 화면의 **'헤더 및 브랜드 설정'** 카드에도 사이트명 입력창(`input-siteName`)을 추가하여 탭 클릭 모달 외에도 메인 편집 폼에서 사이트명을 직접 수정할 수 있도록 활성화.
    - 활성화된 사이트 탭(각 사이트의 포인트 컬러 동적 적용)을 전환할 때마다 3depth 메뉴 트리, 브랜드 에디터(로고, 테마 컬러, CS정보 등)가 해당 사이트의 설정 데이터로 즉시 교체 및 개별 저장.
- **메뉴별 상세 등급 노출 설정 및 신규 사이트 등급 선택의 Shuttle Box (Dual List Box) 전환**:
    - **셔틀 박스 UI 컴포넌트 탑재**: '신규 사이트 추가' 모달 및 '메뉴 노출 조건 설정' 모달 내의 등급 선택 UI를 최신 셔틀 박스(Dual List Box) 형태로 전면 개편.
    - **다차원 실시간 검색 및 다중 선택**: 좌/우 각 패널 상단에 등급 검색창(`filterLeft/Right`)을 배치하여 등급이 많을 시에도 100ms 이내 초고속 필터링 지원, 개별 다중 체크박스를 통한 편리한 추가/제거 기능 구현.
    - **중앙 직관적 액션 컨트롤**: `[추가 ▶]`, `[◀ 제거]` 버튼을 통해 선택된 등급군을 실시간으로 상태 이동 및 동적 렌더링.
    - **데이터 격리 예외 처리**: 사이트 간 완벽한 데이터 독립성(Data Isolation)을 위해, 이미 다른 사이트에 매핑 완료된 등급은 좌측 목록에서 취소선과 함께 비활성화(Disabled) 및 `(다른 사이트 지정됨)` 배지를 표시하고 중복 추가를 원천 차단하여 비즈니스 논리 안정성 극대화.
    - **오브젝트 출력 방지**: 기존 등급 데이터 로드 시 등급 속성 자료형(String/Object)에 따른 `[object Object]` 문자열 출력 문제를 감지하여 완벽히 방어(Fallback 로직 구현 완료).
- **사용자 포털 브릿지 페이지 (Bridge Selection Page) 구현**:
    - 로그인한 사용자가 2개 이상의 사이트에 접근 권한이 있을 경우, 진입 전 맞춤형 포털 목록을 보여주는 **고품격 사이트 선택 브릿지 카드 UI** 구현.
    - 카드 호버 시 상단 테마 컬러 바가 강조되며 부드럽게 떠오르는 그림자(Shadow)와 버튼 전환 효과 등 프리미엄 디자인 설계.
    - 단 1개의 사이트에만 접근 가능한 유저의 경우, 브릿지 페이지를 스킵하고 즉시 해당 포털 메인으로 자동 포워딩하여 사용자 편의성(UX) 극대화.
- **포털 내 실시간 사이트 스위처 Dropdown 적용**:
    - 다중 사이트 접근 권한을 가진 사용자가 포털 내부에서 로그아웃 없이 다른 사이트로 즉시 전환할 수 있도록, 헤더 우측 사용자명 옆에 테마 컬러가 입혀진 **실시간 사이트 셀렉터 Dropdown** 탑재.
- **Fine-Grained 재귀적 메뉴 권한 필터링**:
    - 어드민에서 특정 메뉴에 설정한 노출 등급 세팅값(`menu.exposedTiers`)을 재귀적으로 탐색하여 유저 등급과 비교 분석(합집합 OR 조건).
    - 권한이 없는 메뉴는 GNB 및 사이드바 DOM 렌더링 단계에서 완벽하게 차단/제거하여 보안 및 정보 접근 통제 완비.

### '병원 및 명의안내' 메뉴 내 '명의안내' 및 '병원검색' 서브페이지 고도화
- **명의안내 (expert) 고도화**:
    - 명의안내 프로세스(전화상담 신청 ➔ 간호사 상담 ➔ 증상 및 상태 확인 ➔ 병원 및 의료진 안내)를 상징하는 원형 4단계 프리미엄 디자인 카드 레이아웃 구현.
    - 하단의 `상담 신청하기 >` 클릭 시 부드럽게 열리는 `전문가 상담 신청 폼` 연동 및 신청 정보 제출 로직 완성.
- **병원검색 (search) 고도화**:
    - **다차원 상세 주소 검색 필터**: `[주소별 검색]` / `[도로명 검색]` 계층형 탭 필터, 시도/시군구/읍면동 선택기 및 `[ ] 현재 위치로 검색` 체크 제어 탑재.
    - **체크박스 분류 매트릭스 (3단)**: 진료과목 선택(정형외과 등 16종 + 검색 기능), 특정 분야 선택(관절센터 등 12종), 병원 종별 선택(상급종합병원 등 8종) 체크박스 리스트 적용 및 정형외과/관절센터/상급종합병원 기본 체크 적용.
    - **인터랙티브 HTML/CSS 벡터 맵 모형 (Mockup)**: 격자 도로망, 연두색 공원, 한강(Han River) 물결 밴드가 조합된 고급 웹 지도 모듈 탑재. 지도 상단에 거리 비례 위치별 번호 마커(1, 2, 3) 및 클러스터링 개수 배지(13, 38, 19), 줌 조절 Widget, 축척 레전드 범례 구현.
    - **검색 결과 & 연계 동작 고도화**: 거리별(1.2km 등) 상세 카드 리스트 및 정렬 필터 추가. `상세보기` 클릭 시 상담 신청 탭으로 자동 이동하는 해시 라우팅 연계 완료.

## [2026-05-13]
### 기능 고도화 및 브랜드 관리 시스템 도입
- **3단계 계층형 메뉴 시스템 (3-Level Menu)**:
    - 1depth > 2depth > 3depth 구조의 재귀적 트리 메뉴 관리 기능 구현
    - 각 메뉴 노드별 독립적인 노출 여부(Show/Hide) 및 명칭(기본명/고객사별 명칭) 설정 가능
    - 대규모 메뉴 구조를 한눈에 파악하고 편집할 수 있는 확장형 관리 UI 적용
- **프리미엄 메가메뉴 (Mega-Menu) 구현**:
    - 사용자 포털 헤더 호버 시 3단계 계층 구조가 한눈에 펼쳐지는 그리드 기반 메가메뉴 레이아웃 적용
    - 부드러운 애니메이션 및 직관적인 다단 컬럼 네비게이션 제공
- **사이드바 아코디언 네비게이션**: 3단계 계층 구조를 지원하며, 선택 메뉴의 하위 항목이 자동 확장되는 직관적인 트리뷰 구현
- **1:1 온라인 문의 시스템**: 사용자 개인 질문과 전문가 답변을 매칭하는 비공개 문의 게시판 및 어드민 대응 시스템 구축
- **헤더 및 브랜드 통합 관리 (Branding)**:
    - **로고 업로드 시스템**: SVG 및 이미지 파일을 관리자 페이지에서 직접 업로드하여 실시간 반영 (Base64 저장)
    - **2포인트 컬러 시스템**: 포인트 컬러 1(버튼/로그인 배경)과 포인트 컬러 2(메뉴/CS번호)로 구성된 직관적인 테마 관리
    - **HEX 코드 직접 입력 및 미리보기**: 컬러 피커 대신 HEX 코드를 직접 입력하고 즉시 색상을 확인하는 프리뷰 기능 탑재
- **동적 푸터(Footer) 및 정책 관리**:
    - 고객사명, 고객사 링크, 서비스 제공사명(교보다솜케어), 제공사 링크 동적 설정 기능
    - 공식 서비스 고지 문구(Disclaimer) 및 법적 필수 링크(개인정보처리방침, 이용약관) 연동
- **글로벌 브랜딩 기본값(Default) 수립**:
    - 모든 신규 고객사 포털에 대해 '교보다솜케어' 브랜딩을 기본값으로 강제 적용하여 초기 설정 편의성 및 브랜드 통일성 강화

- **동적 사이드바 네비게이션 구현**: 100개 이상의 사용자 등급을 처리할 수 있는 등급 코드 등록 시스템 및 재귀적 트리 설정 UI를 통한 등급별 가시성 제어 기능 구현
- **통합 버그 수정**: `app.js`의 코드 중복 및 구문 오류 해결을 통해 동적 메뉴 구성과 관리자 페이지의 등급별 가시성 설정이 정상적으로 연동되도록 수정
